PORTFOLIO | JOB SCHIEVEN | 2022



Ervaring connection

Tijdens de workshops van "Smart Connection" zijn we bezig geweest met het kennis maken met het bouwen van een website en alles wat hiermee te maken heeft Ik heb in mijn leven natuurlijk veel gebruik gemaakt van het internet en daarmee veel verschillende sites bezocht. Altijd was ik al eens benieuwd hoe zoiets tot stand komt en wat nou het verschil is tussen een mooie en een simpele site. Tevens was ik nieuwsgierig of het enigszins lijkt op het ontwikkelen van software zoals ik dit op mijn opleiding en voor mijn werk doe.

In eerste instantie zijn we dieper ingegaan op onze huidige kennis. Als opdracht hiervoor konden we een model opstellen hoe wij dachten dat verschillende onderwerpen aan elkaar verbonden waren. Hierin ging het over onderwerpen als:
  • HTML, CSS & JavaScript
  • API
  • SQL/NoSQL & Cloud Database
  • Machine Learning & AI
  • Internet Of Things

 De uitwerking van mijn verbindingsmodel staat hieronder weergegeven. Ik heb deze opgedeeld in 2 delen. Het eerste model heeft meer betrekking tot website interface en de tweede meer over de data stroom.


Workshops

WebDevelopment
Na kennis gemaakt te hebben met wat onderwerpen die aan bod komen bij het ontwikkelen van een site zijn we dieper ingegaan op de werking zelf Hierin hebben we gekeken naar de opbouw van een site door wat rond te neuzen op het internet en door wat voorbeelden. Door middel van de voorbeelden werd mij al gauw duidelijk welke impact bepaalde functies of veranderingen in de code hadden. Hierin merkte ik wel een voordeel omdat ik vanuit de opleiding al veel bezig geweest ben met het ontwikkelen van code. Daarentegen is dit wel totaal anders en was het ook juist weer moeilijker daardoor omdat ik opzoek ging naar vergelijkingsmateriaal terwijl het helemaal niet te vergelijken is.

Site opmaak en algoritmes
Met wat beginkennis zijn we begonnen aan het maken van onze portfolio. Door het zelf toe te passen is het een stuk eenvoudiger te begrijpen. Voor mij heeft dit weer benadrukt dat al doende leert men. Witek ten Hove heeft enkele voorbeelden laten zien hoe je bepaalde toepassingen kunt invoegen in een site. Zoals een teachable bot van google die objecten herkent doormiddel van afbeeldingen. Heel erg gaaf om hierin de verbinding te zien met smart industry en dat een machine learning algoritme zo eenvoudig is toe te passen

Database
Vanuit mijn werk was ik bekend met de term SQL. Voor het opslaan van receptgegevens in de procestechniek wordt een SQL database erg veel toegepast. Echter was ik niet bekend met NoSQL. Ik vind het altijd leuk dat dingen een keerzijde hebben en dat mensen andere meningen dan elkaar hebben en dat hieruit dat dan verschillende uitkomsten bied. Zo is NoSQL ook een afgeleide van SQL maar toch het tegenovergestelde. Het intersante aspect vondt ik de geheugen besparing die NoSQL heeft ten opzicht van SQL. En dat alleen door de data op een andere manier te structureren.

API
Ik had al eens iets gehoord over de term API (Application Programming Interface), maar wat deze inhoud had ik eigenlijk geen idee van. interresant om te leren hoe sites/applicaties informatie uitwisselen met elkaar of data van elkaar gebruiken middels een API. Ik heb geconcludeerd dat dit een erg machtige interface is omdat het zo veel wordt toegepast en hier heel veel op in is gericht. Ook heb ik kunnen leren wat het verschil is tussen een forend API en een backend API. Naar mijn idee waren dit 2 verschillende soorten API's maar ik heb geleerd dat de forend de client is en de backend de database is. En dat beide wel een eigen stuk software hebben die dit regelen

Resultaat
Het resultaat van deze lessen is dit Portfolio! Door de opgedane kennis en met hulp van medestudenten is het mij gelukt om een eigen website te bouwen . Erg gaaf om dit te kunnen doen en dat het zo laagdrempelig is om te doen.
